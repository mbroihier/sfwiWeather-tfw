Test Case INIT-0020
Title Installation of the weather server on an existing PI running Stretch
Objective

This test case is intended to verify the following implied or specified requirements:

  • When the server is running, it shall report runtime and communication errors to a log file viewable by an administrator.
  • If the server successfully initializes, it shall report to the log that it is listening to a HTTP port.

This test case illustrates the that repository can be installed on an existing PI running Stretch. In this case, the installation will deviate from the repository that creates an "appliance". This illustrates how the server can be run in a mode that is useful for debugging.
Setup
  • Existing PI with about 4 Gig of spare memory running Raspbian Stretch.
Actions
  1. Log onto the PI account of the preinstalled PI.
  2. Ignore the GitHub.com/mbroihier/sfwiWeather installation instructions up to step 5.
  3. Execute instructions 5 to 14.
  4. Start the weather server using the command: node weather.js
  5. Verify that the server reports that it is listening.
  6. Verify that no errors are reported.
  7. Using a browser on another computer in the local network, connect to the server by using the node name of the sever and the port the server is listening on (e.g. http://rpiOther:3000).
  8. Verify a page is displayed in the browser that is the expected location being reported by the server.
Expected The server will come up when commanded and display that it is listening and will not display any errors. A HTTP client will be able to query the server and a page consistent with the configured location will be displayed.
Results PASSED
Cleanup
Requirements SFWIWEATHER-PR-INIT-010, SFWIWEATHER-PR-INIT-020
