Test Case INIT-0010
Title Install repository on a fresh Raspberry PI
Objective

This test case is intended to verify the following implied or specified requirements:

  • sfwiWeather server shall be configured prior to starting for the purpose of collecting weather information for a location within the United States. That configuration will include:
  • sfwiWeather shall be configured with a latitude and longitude within the United States. Units of latitude and longitude will be degrees, positive will be Northern Hemisphere and Eastern Hemisphere.
  • sfwiWeather shall be configured with a location name.
  • The application shall be able to run on a Raspberry PI 0 with 8 gigabytes or less of memory on a version of Raspbian Linux Stretch or above. Note that 8 gigabytes was selected because it is the smallest practical SD card to purchase for the Raspberry PI.
  • The application shall be started automatically as a service after it has been configured and installed via the installation instructions. Note that in this mode of operation, the log file will be sent to /dev/null to prevent excess writing to the SD card.

This test case verifies the installation procedure in the repository "readme" file. Most of what is being tested here are implied requirements. That is, upon completion of installation, the server should run without error and display information to a client.
Setup
  • Buster Raspbian image is required.
  • 8 Gig or more SD card (contents will be overwritten during install)
  • PI 0 W
Actions
  1. Follow the instructions in the readme file in the github.com repository /mbroihier/sfwiWeather. When configuring, use a location of your choice within the US.
  2. If errors are stated during the configuration and the location should have been a valid US coordinate and there is a radar station supporting that location, fail this test case.
  3. If configuration passed, a browser given the sfwiWeather:3000 URL displays weather information and it looks correct/reasonable, pass this test.
Expected The PI will boot and start a server listening to the port 3000 of node sfwiWeather on a private network.
Results PASSED
Cleanup
Requirements SFWIWEATHER-EXT-IF-030, SFWIWEATHER-EXT-IF-030.001, SFWIWEATHER-EXT-IF-030.002, SFWIWEATHER-PROV-010, SFWIWEATHER-PROV-020
