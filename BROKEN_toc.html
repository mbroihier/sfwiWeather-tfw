Broken API Responses

This test category confirms proper operation of the server when the API responds with invalid or unexpected information. This series of test cases was developed by hand and is meant to be run with the test driver using the command "node driver.js BROKEN" from within the sfwiWeather-tfw directory.

  1. BROKEN-0000 - Test category setup
  2. BROKEN-0010 - All good API responses
  3. BROKEN-0020 - Blank forecast response
  4. BROKEN-0030 - No periods field in forecast response
  5. BROKEN-0040 - No windspeed field in forecast response
  6. BROKEN-0050 - No wind direction field in forecast response
  7. BROKEN-0060 - No start time field in forecast response
  8. BROKEN-0070 - No short forecast field in the forecast response
  9. BROKEN-0080 - No features field in an alert response
  10. BROKEN-0090 - No properties field in an alert response
  11. BROKEN-0100 - No headline field in an alert response
