Test Case UPDATE-0010
Title Confirm the Update Frequency and the Use of HTTPS to query the API
Objective

This test case is intended to verify the following implied or specified requirements:

  • sfwiWeather shall collect weather information only from public sites available via the world wide web.
  • sfwiWeather shall collect weather information only using secure interfaces (HTTPS).
  • The server shall collect information to be presented on a client HTTP query GET.
  • The collection interval shall be less than half the update interval of the data being collected for those interfaces updated periodically (eg forecast).
  • Asynchronous updates (eg alerts) shall be collected on demand.

This test case confirms that the server collects weather data from the API at appropriate rates and that HTTPS is used to collect the data.
Setup
  • INST-0020 must have been run prior to this test case.
Actions
  1. Using tcpdump on the server computer, monitor queries to api.weather.gov.
    • tcpdump -s0 -w update-0010.log.raw -c 20000
  2. api.weather.gov forecasts should be updated every hour, so make sure the queries occur at least every 1/2 hour.
  3. Verify the queries are over the https port (443).
Expected The API queries will use HTTPS and they will be faster than 2x the update rates of the forecast information and on demand for the alert data.
Results FAILED
Cleanup
Requirements SFWIWEATHER-EXT-IF-040, SFWIWEATHER-EXT-IF-040.001, SFWIWEATHER-PR-INFO-010, SFWIWEATHER-PR-INFO-020, SFWIWEATHER-PR-INFO-030
